```

├── img/                    # Pasta de imagens
│   └── Logo.png           # Logo da empresa

├── style.css              # Estilos CSS
```

## 🛠️ Comandos Disponíveis

### Desenvolvimento
```bash
npm start       # Inicia servidor de desenvolvimento (porta 3000)
npm run dev     # Alias para npm start
```

### Build e Produção
```bash
npm run build   # Gera pasta dist/ otimizada
npm run clean   # Remove pasta dist/
npm run rebuild # Limpa e reconstrói
```

### Executar Produção
```bash
cd dist && node server.js
```

## 🔧 Tecnologias
- **Frontend**: HTML5, CSS3 puro
- **Backend**: Node.js nativo (sem frameworks)
- **Servidor**: HTTP nativo do Node.js
- **Porta**: 3000

## 📦 Processo de Build
O script `build.js` realiza:
1. Criação da pasta `dist/`
2. Minificação do CSS
3. Otimização do HTML
4. Cópia de imagens
5. Cópia do servidor
6. Geração de `package.json` simplificado
7. Criação de `build-info.json` com metadados

## ✨ Melhores Práticas

### Ao Modificar o Projeto:
1. **Edite sempre os arquivos na raiz** (não em dist/)
2. **Execute `npm run build`** após alterações
3. **Teste no navegador** antes de commitar
4. **Mantenha o design minimalista** e clean
5. **Use apenas Node.js nativo** (sem dependências)

### Padrões de Código:
- **HTML**: Semântico, indentação de 4 espaços
- **CSS**: Mobile-first, use flexbox para layout
- **JavaScript**: ES6+, código limpo e comentado
- **Cores**: Preto (#000000) e Branco (#FFFFFF)

### Estrutura de Commits:
- `feat:` para novas funcionalidades
- `fix:` para correções
- `style:` para mudanças de estilo
- `docs:` para documentação
- `build:` para mudanças no build

## 🚀 Deploy
